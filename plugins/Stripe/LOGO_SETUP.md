# Stripe 支付页面 Logo 配置指南

## 🎨 配置 Logo 的两种方式

### 方式 1：通过 Stripe Dashboard 配置（全局配置，推荐）

这是最简单且推荐的方式，配置一次后所有 Stripe 支付页面都会使用。

#### 步骤：

1. **登录 Stripe Dashboard**
   - 访问：https://dashboard.stripe.com/
   - 使用您的 Stripe 账号登录

2. **进入品牌设置**
   - 点击左侧菜单的 **Settings**（设置）
   - 找到并点击 **Branding**（品牌设置）

3. **上传 Logo 和图标**
   - **Icon**（图标）
     - 建议尺寸：128x128 像素
     - 格式：PNG、JPG
     - 用途：显示在浏览器标签页和部分界面

   - **Logo**（品牌 Logo）
     - 建议尺寸：512x512 像素或更大
     - 格式：PNG（支持透明背景）
     - 用途：显示在 Checkout 页面顶部

4. **设置品牌颜色（可选）**
   - **Brand color**（品牌主色）：按钮和链接的颜色
   - **Accent color**（强调色）：次要元素的颜色

5. **保存配置**
   - 点击 **Save**（保存）
   - 配置立即生效，所有新创建的 Checkout 会话都会使用新的品牌设置

#### 优点：
✅ 配置一次，全局生效
✅ 无需修改代码
✅ Stripe 官方推荐方式
✅ 自动处理图片缓存和优化

---

### 方式 2：通过插件配置 Logo URL（动态配置）

如果您需要为不同的订单或商品显示不同的 Logo，可以使用这种方式。

#### 步骤：

1. **准备 Logo 图片**
   - 上传您的 Logo 到服务器或图床
   - 必须使用 **HTTPS** 链接（HTTP 不支持）
   - 建议尺寸：**512x512 像素**（正方形）
   - 推荐格式：PNG（支持透明背景）

2. **在插件配置中填写 Logo URL**
   - 登录 Xboard 管理后台
   - 进入插件管理，找到 **Stripe 支付** 插件
   - 在配置页面找到 **Logo URL** 字段
   - 填入您的 Logo 图片完整 URL，例如：
     ```
     https://yourdomain.com/assets/logo.png
     ```

3. **保存并测试**
   - 保存插件配置
   - 在服务器执行：`php artisan octane:reload`
   - 创建一个测试订单，查看支付页面是否显示新 Logo

#### 支持的图片要求：
- ✅ 必须是 HTTPS 链接
- ✅ 图片格式：PNG、JPG、JPEG
- ✅ 建议尺寸：512x512 像素（最大 1024x1024）
- ✅ 文件大小：建议小于 500KB
- ❌ 不支持 HTTP 链接
- ❌ 不支持 Base64 编码
- ❌ 不支持 SVG 格式

#### 优点：
✅ 可以动态设置不同的 Logo
✅ 无需访问 Stripe Dashboard
✅ 适合多品牌或多商户场景

## 🔧 常见问题

### Q1: Logo 没有显示，还是显示企业名称？

**A:** 可能的原因：

1. **Logo 图片不符合要求**
   - 检查是否使用 HTTPS 链接
   - 确认图片大小不超过 500KB
   - 使用 PNG 或 JPG 格式

2. **缓存问题**
   - 清除浏览器缓存
   - 在服务器执行：`php artisan cache:clear`
   - 重新加载 Octane：`php artisan octane:reload`

3. **Stripe Dashboard 的优先级**
   - Stripe Dashboard 的 Branding 设置优先级最高
   - 如果 Dashboard 中已配置 Logo，插件配置的 Logo 可能不生效

### Q2: 如何隐藏左上角的企业名称？

**A:** 企业名称来自 Stripe 账户设置，无法完全隐藏，但可以通过以下方式优化：

1. **修改 Stripe 账户的企业名称**
   - Stripe Dashboard > Settings > Account details
   - 修改 **Business name** 为您的品牌名称

2. **上传 Logo**
   - 配置 Logo 后，会在企业名称旁边显示 Logo 图标
   - 用户会更关注 Logo 而不是文字

### Q3: 能否为不同的商品设置不同的 Logo？

**A:** 目前版本不支持商品级别的 Logo 配置，有两种解决方案：

1. **使用插件的 Logo URL 配置**
   - 在创建订单前，动态修改插件配置中的 Logo URL
   - 适合有开发能力的用户

2. **在 Stripe Dashboard 创建多个企业账号**
   - 每个品牌使用独立的 Stripe 账号
   - 为每个账号配置不同的 Logo

### Q4: Logo 显示变形或模糊怎么办？

**A:** 优化建议：

1. **使用正方形图片**
   - 推荐尺寸：512x512 像素
   - 避免使用矩形图片

2. **使用高分辨率图片**
   - 最小：256x256 像素
   - 推荐：512x512 像素
   - 最大：1024x1024 像素

3. **优化图片格式**
   - 使用 PNG 格式（支持透明背景）
   - 使用图片压缩工具减小文件大小
   - 推荐工具：TinyPNG (https://tinypng.com)

### Q5: 测试环境和生产环境的 Logo 配置

**A:** 配置方法相同，但注意：

- **Stripe Dashboard 方式**：测试模式和生产模式使用相同的 Branding 设置
- **插件配置方式**：可以为测试环境和生产环境配置不同的 Logo URL

## 🎯 最佳实践

### 推荐方案：组合使用两种方式

1. **在 Stripe Dashboard 配置基础 Logo**
   - 设置默认的品牌 Logo
   - 配置品牌颜色
   - 作为全局默认配置

2. **在插件配置中设置特定 Logo（可选）**
   - 为特殊商品或活动设置专属 Logo
   - 在代码中动态修改配置

这样既保证了全局一致性，又具有灵活性！

## 💡 Logo 设计建议

### 适合支付页面的 Logo 特点：

✅ **简洁明了**
- 避免过于复杂的图案
- 清晰可辨的品牌标识

✅ **高对比度**
- 确保在浅色背景上清晰可见
- 使用深色 Logo 或带轮廓的设计

✅ **可识别性**
- Logo 应该与您的品牌一致
- 用户一眼就能识别

✅ **专业性**
- 使用高质量的设计
- 避免像素化或模糊
